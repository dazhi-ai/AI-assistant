# AI助手开发任务拆解与环境准备（v1.0）

## 1. 开发总览
- 开发模式：先打通最小闭环，再逐步增强能力与安全。
- 闭环定义：用户发起指令 -> 后端理解意图 -> 执行插件动作 -> 平板端收到反馈并展示。
- 阶段顺序：后端中枢 -> 平板前端 -> 硬件接入 -> 联调测试 -> 部署上线。

## 2. 任务拆解（按阶段）

## 阶段 A：后端中枢（优先级 P0）
### A1. 配置与基础框架
- 目标：完成可运行的 Python 服务骨架与配置体系。
- 任务：
  - 完成 `.env` 参数加载、默认值与启动校验。
  - 建立模块结构：`src/`、`plugins/`、`tests/`。
  - 增加统一日志模块（info/error/request_id）。
- 完成标准：
  - `python main.py` 可启动 WebSocket 服务。
  - 缺失关键配置时可输出明确错误信息并退出。

### A2. WebSocket 网关协议
- 目标：定义前后端统一消息协议并实现基础收发。
- 任务：
  - 定义消息格式：`type`、`trace_id`、`payload`、`timestamp`。
  - 定义核心事件：`TEXT`、`AUDIO_CHUNK`、`MODEL_SWITCH`、`EFFECT`、`ERROR`。
  - 增加心跳：`PING/PONG` 与超时断线处理。
- 完成标准：
  - 平板可稳定连接并接收文本与控制事件。
  - 空闲状态 30 分钟不断线（内网环境）。

### A3. 豆包模型接入与工具调用解析
- 目标：实现 Function Calling 执行闭环。
- 任务：
  - 接入 Ark（OpenAI 兼容）聊天接口。
  - 实现 `tool_calls` 解析分发器（Function Call Handler）。
  - 根据任务类型动态调整 Temperature。
- 完成标准：
  - 输入“帮我搜周杰伦的歌”，可触发 `search_music` 工具链路。
  - 多工具调用顺序可控，异常可回传前端。

### A4. 网易云插件（P0 核心）
- 目标：完成音乐能力最小可用集。
- 任务：
  - 封装 `NeteaseCloudController`：登录二维码、搜索、播放、点红心、收藏、歌单。
  - 统一错误处理：未登录、无结果、接口失败、网络超时。
  - 与 WebSocket 联动推送视觉反馈（例如 HEART 动效）。
- 完成标准：
  - 指令“好听”可触发点红心。
  - 指令“收藏这首歌”可触发收藏，不与点红心混淆。

## 阶段 B：平板前端容器（优先级 P1）
### B1. Live2D 基础渲染
- 目标：在小米平板端稳定展示模型。
- 任务：
  - 使用 PixiJS + Live2D 完成模型加载。
  - 实现模型资源按配置切换（JSON 驱动）。
- 完成标准：
  - 至少 2 套模型可在运行时切换成功。

### B2. 语音与口型联动
- 目标：实现 TTS 音频与口型同步。
- 任务：
  - Web Audio API 分析音频能量并映射 Mouth Open。
  - 前端实现音频缓冲队列，减少爆音与断续。
- 完成标准：
  - 连续 20 条短句播放无明显卡顿。

### B3. 兼容性与连接稳定
- 目标：兼容老安卓内核并提升稳定性。
- 任务：
  - Babel 编译至 ES5。
  - 心跳与自动重连（指数退避）。
  - 降级方案：低性能模式（减少动画特效）。
- 完成标准：
  - 平板静置后恢复可自动重连，不需手动刷新页面。

## 阶段 C：硬件接入与联调（优先级 P1）
### C1. ESP32 接入
- 目标：将语音输入源切换为小智硬件。
- 任务：
  - 烧录固件并指向自建后端地址。
  - 打通语音采集 -> 后端意图处理 -> 反馈播报。
- 完成标准：
  - 语音指令端到端平均响应可接受（主观体验流畅）。

### C2. 全链路联调
- 目标：验证“语音 + 音乐 + 视觉 + 天气”完整流程。
- 任务：
  - 设计 20 条高频指令测试集。
  - 对异常场景做回归（断网、未登录、第三方接口超时）。
- 完成标准：
  - 关键路径通过率 >= 95%。

## 阶段 D：上线与运维（优先级 P1）
### D1. 部署与守护
- 目标：服务可持续运行并自动恢复。
- 任务：
  - 使用 systemd 落地后端与网易云 API 进程守护。
  - 配置开机自启、日志滚动与健康检查。
- 完成标准：
  - 服务器重启后 2 分钟内服务自动可用。

### D2. 安全基线
- 目标：满足最小安全上线要求。
- 任务：
  - `.env` 脱敏与仓库忽略校验。
  - WebSocket token 校验。
  - 网易云 API 本地端口隔离。
- 完成标准：
  - 安全自检清单全部通过。

## 3. 本周建议执行顺序（你可以直接照此推进）
1. 完成 A2（WebSocket 协议）和 A3（工具调用分发器）基础代码。
2. 完成 A4 网易云插件最小能力（登录二维码 + 搜索 + 点红心）。
3. 前端完成 B1 模型展示和 B3 连接稳定，再做 B2 口型同步。
4. 最后接入硬件并做 C2 全链路回归。

## 4. 开发环境准备清单

### 4.1 本地基础环境
- Python：3.10 或以上
- Node.js：18 或以上（平板前端构建用）
- Git：最新版
- systemd（服务器侧进程守护，Linux 环境）

### 4.2 Python 侧准备
- 创建虚拟环境并激活：
  - `python -m venv .venv`
  - Windows：`.venv\Scripts\activate`
- 安装依赖：
  - `pip install -r requirements.txt`
- 环境变量：
  - 复制 `.env.example` 为 `.env`
  - 填入 `ARK_API_KEY`、`NETEASE_API_URL`、`WS_TOKEN` 等关键参数

### 4.3 前端侧准备（平板页面）
- 初始化前端工程（建议独立目录 `web-client/`）。
- 安装依赖：`pixi.js`、Live2D 相关库、构建工具（Vite/Webpack）。
- 配置 Babel 输出目标 ES5。

### 4.4 联调工具
- WebSocket 调试工具（如 Postman / Apifox / websocat）
- 抓包/日志工具（用于排查断流、重连、音频分片）

## 5. 接口约定（第一版草案）

### 5.1 WebSocket 消息结构
```json
{
  "type": "TEXT",
  "trace_id": "uuid",
  "timestamp": 1730000000,
  "payload": {}
}
```

### 5.2 建议事件类型
- `TEXT`：文本消息
- `AUDIO_CHUNK`：音频分片（二进制或 base64）
- `MODEL_SWITCH`：切换 Live2D 模型
- `QRCODE`：展示登录二维码
- `EFFECT`：视觉效果（例如 HEART）
- `ERROR`：错误信息
- `PING` / `PONG`：心跳

- 平板断网恢复后可自动重连并继续收消息。
