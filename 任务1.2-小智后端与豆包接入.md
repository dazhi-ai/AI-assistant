# 任务 1.2 完成说明：部署小智后端并接入豆包模型

> **版本**：v2.0（已更新 ASR/TTS 为火山引擎全 API 方案，移除本地模型依赖）

---

## 0. 方案定义

| 项目 | 选型 | 原因 |
|---|---|---|
| LLM | 豆包（DoubaoLLM，火山引擎 Ark） | OpenAI 兼容，稳定低延迟 |
| ASR | 火山引擎 DoubaoASR | API 调用，无需运行 FunASR 本地模型 |
| TTS | 火山引擎 HuoshanDoubleStreamTTS | 双流式，比 EdgeTTS 快约 2.5s，API 调用 |
| 管理后台 | xiaozhi-esp32-server 智控台 | 设备管理、OTA 升级、配置、多用户 |

**明确拒绝的方案**：FunASR（本地）、GPT-SoVITS（本地）、SherpaASR（本地）——所有需要本地运行模型的 ASR/TTS 均不采用，服务器资源优先保证 API 调用链路。

## 1. 前提条件

### 1.1 火山引擎账号开通
1. 登录 https://console.volcengine.com
2. 开通「语音识别（ASR）」服务：  
   控制台 -> 语音技术 -> 语音识别 -> 开通服务
3. 开通「语音合成（TTS）」服务：  
   控制台 -> 语音技术 -> 语音合成 -> 开通服务  
   ⚠️ 开通后约 30 分钟生效，请耐心等待
4. 创建应用（语音服务）：  
   控制台 -> 语音技术 -> 应用管理 -> 创建应用  
   获得 `appid` 和 `access_token`（即 `VOLC_APP_ID` / `VOLC_ACCESS_TOKEN`）
5. 创建 Ark API Key：  
   控制台 -> 方舟 -> API 密钥管理 -> 创建 API Key（即 `ARK_API_KEY`）

### 1.2 服务器要求
- **最低配置（全 API 方案，无本地模型）**：2 核 2G，推荐 2 核 4G
- 系统：Ubuntu 20.04+ 或 CentOS 7+（含 systemd）
- 已安装：git、Python 3.10+（conda 推荐）

## 3. 最简化安装（仅 Server，无智控台，适合快速验证）

### 3.1 克隆源码
```bash
cd /opt
git clone https://github.com/xinnan-tech/xiaozhi-esp32-server.git
```

### 3.2 使用一键脚本配置
```bash
# 替换以下参数后执行
bash /opt/ai-assistant/scripts/xiaozhi/deploy-xiaozhi-linux.sh \
    /opt/xiaozhi-esp32-server \
    你的ARK_API_KEY \
    你的VOLC_APP_ID \
    你的VOLC_ACCESS_TOKEN \
    你的服务器局域网IP
```

脚本完成后会自动生成 `/opt/xiaozhi-esp32-server/main/xiaozhi-server/data/.config.yaml`，内容包含：
- `DoubaoLLM`（豆包）
- `DoubaoASR`（火山 ASR API）
- `HuoshanDoubleStreamTTS`（火山流式 TTS API）

### 3.3 手动配置（可选，不使用脚本）
参考模板文件 `scripts/xiaozhi/.config.yaml.volc-api-only.example`，填入密钥后放置到：  
`/opt/xiaozhi-esp32-server/main/xiaozhi-server/data/.config.yaml`

### 3.4 启动验证
```bash
conda activate xiaozhi-esp32-server
cd /opt/xiaozhi-esp32-server/main/xiaozhi-server
python app.py
```

日志出现以下关键字即为启动成功：
```
OTA  地址: http://<server-ip>:8003/xiaozhi/ota/
WS   地址: ws://<server-ip>:8000/xiaozhi/v1/
```

## 4. systemd 托管（最简化模式）

### 4.1 服务文件
模板路径：`scripts/systemd/xiaozhi-server.service`

### 4.2 安装命令
```bash
sudo cp /opt/ai-assistant/scripts/systemd/xiaozhi-server.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable xiaozhi-server
sudo systemctl start xiaozhi-server
sudo systemctl status xiaozhi-server
```

## 5. 与本项目（Python AI-assistant）联动

本项目 Python 后端（`main.py`）与 `xiaozhi-esp32-server` 定位不同：

| 服务 | 职责 |
|---|---|
| `xiaozhi-esp32-server` | 处理 ESP32 硬件协议（OTA、WebSocket 硬件握手）、智控台管理 |
| 本项目 `main.py` | 平板端 WebSocket 网关、网易云、天气、Live2D 指令分发 |

建议链路：
- 硬件语音 → xiaozhi-server → 豆包 LLM → 回复给硬件
- 平板控制 → 本项目 main.py → 豆包 LLM → 网易云/天气/Live2D
