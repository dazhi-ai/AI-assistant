# 任务 3.1：ESP32 固件接入自建服务器指引

> 当前：服务端（智控台 + xiaozhi-server）已部署且测试正常，下一步为固件配置/烧录。

---

## 一、两种接入方式（二选一）

| 方式 | 难度 | 适用场景 |
|------|------|----------|
| **方式 A：预编译固件 + 设备内配置** | ⭐ 简单 | 使用虾哥 1.6.1+ 固件，设备支持「高级选项」内填 OTA 地址 |
| **方式 B：自行编译固件** | ⭐⭐ 需 ESP-IDF | 修改源码默认 OTA 地址、定制功能 |

---

## 二、方式 A：使用虾哥预编译固件（推荐）

### 1. 下载固件
- 打开：https://github.com/78/xiaozhi-esp32/releases
- 下载 **1.6.1 及以上** 版本的固件（如 `xiaozhi-esp32-v1.6.1.bin`）

### 2. 确认 OTA 接口正常
浏览器访问（将 IP 换成你的服务器局域网 IP）：
```
http://124.223.174.173:8003/xiaozhi/ota/
```
或（全模块智控台）：
```
http://124.223.174.173:8002/xiaozhi/ota/
```

若显示「OTA 接口运行正常，websocket 集群数量：X」，则继续。

若提示不正常，请在智控台 → 参数管理 → `server.websocket` 填入：
```
ws://124.223.174.173:8000/xiaozhi/v1/
```

### 3. 烧录固件（若设备尚未烧录）
- 使用 [ESP-Launchpad](https://espressif.github.io/esp-launchpad/) 或 `esptool` 烧录下载的 bin 文件

### 4. 设备配网并填入 OTA 地址
1. 进入设备 **配网模式**
2. 点击顶部 **「高级选项」**
3. 在 OTA 地址栏填入：`http://124.223.174.173:8003/xiaozhi/ota/`（或 8002，视你实际 OTA 端口）
4. 保存并重启设备

### 5. 验证
唤醒小智，观察串口日志或智控台设备列表，确认设备已连接。

## 三、方式 B：自行编译固件

### 1. 克隆固件源码
```bash
cd D:\AI-assistant
mkdir -p services
cd services
git clone https://github.com/78/xiaozhi-esp32.git xiaozhi-esp32-firmware
cd xiaozhi-esp32-firmware
```

### 2. 修改 OTA 地址
编辑 `main/Kconfig.projbuild`，找到 `OTA_URL` 的 `default`，改为：
```
default "http://124.223.174.173:8003/xiaozhi/ota/"
```
（将 IP 换成你的服务器局域网 IP）

### 3. 配置编译环境
参考：[《Windows 搭建 ESP-IDF 5.3.2 开发环境以及编译小智》](https://icnynnzcwou8.feishu.cn/wiki/JEYDwTTALi5s2zkGlFGcDiRknXf)

### 4. 编译并打包
```bash
cd xiaozhi-esp32-firmware
idf.py set-target esp32s3   # 或 esp32、esp32c3，按板子型号
idf.py menuconfig          # 可选：配置板型 BOARD_TYPE
idf.py build
cd scripts
python release.py
```

生成的固件在 `build/merged-binary.bin`。

### 5. 烧录
使用 [ESP-Launchpad](https://espressif.github.io/esp-launchpad/) 或 `esptool` 烧录 `merged-binary.bin`。
