# 任务 1.3：智控台（Admin 后台）部署与接入指南

> 本文档描述如何部署 `xiaozhi-esp32-server` **全模块**，以获得完整的 Web 管理后台（智控台）。  
> 智控台功能：设备管理、OTA 升级、ASR/TTS/LLM 配置、多用户管理、多智能体管理。

---

## 1. 整体架构（全模块）

```
┌─────────────────────────────────────────────────────────────────┐
│  全模块 xiaozhi-esp32-server                                     │
│                                                                   │
│  ┌────────────────────┐   ┌──────────────────────────────────┐  │
│  │  Python Server     │   │  Java Web Server（智控台 API）   │  │
│  │  port: 8000 (WS)   │   │  port: 8002                      │  │
│  │  port: 8003 (OTA)  │   └──────────────────────────────────┘  │
│  └────────────────────┘   ┌──────────────────────────────────┐  │
│                            │  Vue 前端（智控台界面）          │  │
│  ┌────────────────────┐   │  通过 8002 提供                   │  │
│  │  MySQL 数据库      │   └──────────────────────────────────┘  │
│  │  Redis 缓存        │                                          │
│  └────────────────────┘                                          │
└─────────────────────────────────────────────────────────────────┘
         ↑                                    ↑
  ESP32 硬件连接                       浏览器访问管理后台
  （OTA + WebSocket）                 http://服务器IP:8002
```

## 2. 推荐部署方式：Docker 全模块

> Docker 是最简单可靠的方式，推荐服务器 RAM ≥ 4G。

### 2.1 前提
- 服务器已安装 Docker + Docker Compose
- 安装 Docker 命令（如未安装）：
  ```bash
  curl -fsSL https://get.docker.com | bash -s docker
  sudo systemctl enable docker && sudo systemctl start docker
  ```

### 2.2 下载并执行自动化安装脚本
```bash
# 下载脚本
curl -L -o docker-setup.sh \
    https://raw.githubusercontent.com/xinnan-tech/xiaozhi-esp32-server/main/docker-setup.sh

# 赋权并执行（交互式，按提示选择全模块安装）
chmod +x docker-setup.sh
./docker-setup.sh
```

脚本会引导你选择部署模式，选择**全模块（含智控台）**。

### 2.3 手动 Docker Compose 部署（备选）
若脚本不可用，可手动执行：
```bash
cd /opt
git clone https://github.com/xinnan-tech/xiaozhi-esp32-server.git
cd xiaozhi-esp32-server

# 使用全模块 compose 文件启动
docker compose -f docker-compose_all.yml up -d

# 观察启动日志，等待所有服务就绪（约 1-3 分钟）
docker logs -f xiaozhi-esp32-server-web
```

### 2.4 确认各容器已就绪
```bash
docker ps
# 应看到以下容器均为 Up 状态：
# xiaozhi-esp32-server-server（Python）
# xiaozhi-esp32-server-web（Java/Vue 智控台）
# mysql
# redis
```

## 4. 在智控台中配置 ASR/TTS/LLM

> ⚠️ 全模块安装时，ASR/TTS/LLM 配置**通过智控台界面操作**，  
> 而不是直接修改 `.config.yaml`（智控台会生成 `config_from_api.yaml` 覆盖配置）。

### 4.1 配置 LLM（豆包）
1. 智控台 → 智能体管理 → 选择默认智能体 → 编辑
2. LLM 选项 → 选择 `DoubaoLLM`
3. 填写：
   - Base URL：`https://ark.cn-beijing.volces.com/api/v3`
   - API Key：你的 `ARK_API_KEY`
   - 模型名：`doubao-1-5-pro-32k-250115`

### 4.2 配置 ASR（火山引擎，无本地模型）
1. 智控台 → 系统参数 / 参数管理 → ASR 配置
2. 选择模块：`DoubaoASR`
3. 填写：
   - App ID：你的 `VOLC_APP_ID`
   - Access Token：你的 `VOLC_ACCESS_TOKEN`
   - Cluster：`volcengine_input_common`（中文普通话）

### 4.3 配置 TTS（火山引擎流式，无本地模型）

> **豆包语音合成 2.0**：若已在火山控制台将 TTS 升级为 [豆包语音合成 2.0](https://console.volcengine.com/speech/service/10035)，**必须使用 `HuoshanDoubleStreamTTS`**。  
> ⚠️ 不要使用 `DoubaoTTS`（HTTP V1 接口），豆包 2.0 仅支持 WebSocket 双向流式-V3 协议。  
> 文档：[豆包语音合成 2.0 能力介绍](https://www.volcengine.com/docs/6561/1871062) | [WebSocket 双向流式-V3](https://www.volcengine.com/docs/6561/1329505)

1. 智控台 → 系统参数 / 参数管理 → TTS 配置
2. 选择模块：`HuoshanDoubleStreamTTS`（勿选 DoubaoTTS）
3. 填写：
   - App ID：你的 `VOLC_APP_ID`
   - Access Token：你的 `VOLC_ACCESS_TOKEN`
   - Voice（音色）：`zh_female_wanwanxiaohe_moon_bigtts`
   - Cluster：`volcano_tts`

**豆包语音合成 2.0 常用音色（大模型音色）**：

| 音色 ID | 描述 |
|---|---|
| `zh_female_wanwanxiaohe_moon_bigtts` | 女声·温柔 |
| `zh_female_tianmei_moon_bigtts` | 女声·甜美 |
| `zh_male_M392_bigtts` | 男声·成熟 |
| `BV001_streaming` | 流式默认音色（部分场景可用） |

更多音色见火山文档 [大模型音色列表](https://www.volcengine.com/docs/6561/1257544)。

## 5. 配置 OTA 和 WebSocket 地址

智控台 → 系统参数 → 服务配置：
```
OTA 接口地址:       http://服务器局域网IP:8003/xiaozhi/ota/
WebSocket 接口地址: ws://服务器局域网IP:8000/xiaozhi/v1/
```

ESP32 设备连接 WiFi 后，会先请求 OTA 接口，再由 OTA 下发 WebSocket 地址。  
因此两个地址都必须填写为**服务器的局域网 IP**。

---

## 6. 获取 manager-api secret（Python Server 联动配置）

全模块安装时，Python Server 需要与 Java Web 联动：
1. 智控台 → 参数管理 → 搜索 `server.secret`
2. 复制 secret 值
3. 在 Python Server 的 `data/.config.yaml` 中添加：
```yaml
manager-api:
  url: http://xiaozhi-esp32-server-web:8002/xiaozhi
  secret: 从智控台参数管理中复制的secret值
```
4. 重启 Python Server：
```bash
docker restart xiaozhi-esp32-server-server
```

## 7. OTA 烧录固件配置

若需要重新烧录 ESP32 固件指向自建服务器：
1. 参考 `scripts/xiaozhi/固件接入自建后端说明.md`
2. 将 `CONFIG_OTA_URL` 改为：`http://你的服务器IP:8003/xiaozhi/ota/`
3. 重新编译并烧录

## 8. systemd 托管 Docker Compose（可选）

若需要服务器重启后自动拉起 Docker 容器：
```bash
# 创建 systemd 服务
sudo tee /etc/systemd/system/xiaozhi-all.service > /dev/null <<'EOF'
[Unit]
Description=XiaoZhi ESP32 Server (全模块)
Requires=docker.service
After=docker.service

[Service]
WorkingDirectory=/opt/xiaozhi-esp32-server
ExecStart=/usr/bin/docker compose -f docker-compose_all.yml up
ExecStop=/usr/bin/docker compose -f docker-compose_all.yml down
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable xiaozhi-all
sudo systemctl start xiaozhi-all
```

## 10. 常见问题

**Q: 火山 ASR/TTS 服务调用报错 "service not available"？**  
A: 服务开通后需等待约 30 分钟才能生效，请耐心等待后重试。

**Q: 智控台配置后重启 Python Server，TTS 仍走 EdgeTTS？**  
A: 全模块安装时配置来自 `config_from_api.yaml`，而非 `.config.yaml`。  
请在智控台"参数管理"中确认 TTS 选择为 `HuoshanDoubleStreamTTS`，然后重启 Server。

**Q: 想更换 TTS 音色？**  
A: 在智控台 TTS 配置中修改 `voice` 字段。  
可在火山控制台"语音合成 -> 音色广场"试听后选择音色 ID。

**Q: 豆包语音合成 2.0 的 Cluster 填什么？**  
A: 豆包 2.0 与原有大模型 TTS 共用 `volcano_tts` 集群。若控制台开通的是豆包语音合成 2.0（服务 ID 10035），保持 Cluster=`volcano_tts`、Voice 使用大模型音色即可。

**Q: 报错 `core.providers.tts.doubao error: Expecting value: line 1 column 1 (char 0)`？**  
A: 表示当前使用的是 **DoubaoTTS**（HTTP 一次性合成-V1），而豆包语音合成 2.0 仅支持 **WebSocket 双向流式-V3**。  
**处理方式**：在智控台 → 系统参数 → TTS 配置 → 将模块从 `DoubaoTTS` 改为 `HuoshanDoubleStreamTTS`，填写 App ID、Access Token、speaker（如 `zh_female_wanwanxiaohe_moon_bigtts`），然后重启 `xiaozhi-esp32-server`。

**Q: 服务器 RAM 只有 2G，能跑全模块吗？**  
A: MySQL + Redis + Java + Python 合计约需 2-3G RAM，2G 可能不够稳定。  
建议升级到 4G 或关闭非必要服务。若条件受限，可使用最简化安装（无智控台）。
