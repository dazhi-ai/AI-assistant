# 小智「每日新闻」需求：不部署 RAGFlow 的可行替代方案

## 1. 背景与约束

- **目标**：让小智在聊天中能结合**当天/近期新闻**作答，而不是只依赖模型训练截止知识。
- **约束**：当前小智所在服务器 **内存偏小**（例如约 4GB 级别）时，**不建议**强行部署 RAGFlow 全家桶（Elasticsearch/Infinity、MinIO、RAGFlow 主服务等叠加后极易 **OOM**，导致整机卡顿或 Docker/MySQL 异常）。
- **原则**：**保护线上小智服务优先**；知识增强用「更轻」或「外置」的方式解决。

> 说明：**1258** 若为你方 **SSH 端口**，与 RAGFlow 的 Web 端口（常见 8008）无关；下文不混用。

---

## 2. 方案对比（按推荐顺序）

| 方案 | 优点 | 缺点 | 是否占用小智服务器大内存 |
|------|------|------|--------------------------|
| **A. 外置 RAGFlow（另一台 8GB+ 机器或云主机）** | 与智控台「知识库=RAGFlow」官方路径一致；可定时 API 写入文档 | 需要第二台资源或费用 | **不占** 小智机大内存（只填 URL + Key） |
| **B. 大模型「联网搜索」能力（若你用的供应商支持）** | 新闻**按问题实时**获取，不依赖每日灌库；部署零增量 | 依赖供应商能力/费用/稳定性；回答风格需调提示词 | **几乎不占** |
| **C. 每日新闻摘要 → 写入智能体「系统提示/人设」** | 实现简单；不装向量库 | 有**长度上限**；适合「简报」不适合海量文档；更新要自动化需接口或人工 | **极低** |
| **D. 每日摘要文件 + 智控台手工/半自动维护** | 最安全、最可控 | 人工步骤多 | **极低** |

---

## 3. 方案 A（推荐）：RAGFlow 放在「别的机器」，小智只连过去

**思路**：小智 + 智控台 **仍在现有机子**；RAGFlow 部署在 **内存 ≥ 8GB（官方常建议 ≥16GB 更稳）** 的另一台 VPS/家用 NAS/办公室机器。

**智控台配置**（与现有文档一致）：

- **模型配置 → 知识库**：`服务地址` 填 `http://<RAGFlow 所在机公网或内网 IP>:<端口>`（须保证 **`xiaozhi-esp32-server` 容器内能访问**该地址，慎用仅宿主机可用的 `localhost`）。
- **API 密钥**：RAGFlow 内生成的 Key。
- **智能体**：在「配置角色」里绑定对应知识库。

**外部定时写新闻**：定时任务跑在 **能访问 RAGFlow API 的机器**上即可（不必是小智机）。

**适合你**：希望长期用「文档库 + 检索」、且愿意多一台资源。

---

## 4. 方案 B（很适合「每日新闻」）：用「联网」代替本地知识库

**思路**：新闻本质是**高频变化**的信息，很多时候 **检索当天网页** 比「昨夜批量入库」更贴切。

**做法**：

1. 在你当前给小智对接的 **LLM 供应商后台**（如火山方舟/其它）查看是否提供 **联网搜索、插件、Tools** 等（名称因厂商而异）。
2. 在智控台里为该智能体配置**合适的系统提示**（例如：回答时事前先基于检索结果、注明信息日期、不确定时说明来源与时间）。
3. **不必**在小智服务器安装 RAGFlow。

**风险/注意**：费用、延迟、以及供应商对「新闻类」内容的合规策略需自行确认。

---

## 5. 方案 C：每日自动生成「新闻简报」文本，塞进智能体提示词

**思路**：用定时任务（可在**任意轻量机器**或 GitHub Actions）：

1. 拉取若干 **RSS/新闻 API**（注意版权与条款）；
2. 用 **云端 LLM API** 压成 1～3 千字「今日要点」；
3. 把简报更新到智控台里该智能体的 **系统提示/角色描述**（若平台支持 API 则脚本化；若无 API 则每日人工粘贴一次）。

**局限**：提示词长度有限，只适合**摘要**，不适合把大量原文当知识库。

**适合你**：预算有限、接受「要点级」时事、不想维护第二套向量检索。

---

## 6. 方案 D：最低技术 — 每日导出 Markdown/PDF，在智控台知识库里手工上传

若不使用 RAGFlow，而智控台**只认 RAGFlow 一类接口**，则此条可能不可用；若你后续版本支持「本地上传且不依赖 RAGFlow」，可再用。

一般更现实的是：**采用 A 或 B**。

---

## 7. 与本仓库 `AI-assistant` 的关系

- 本仓库的 `KNOWLEDGE_HTTP_PORT` / `knowledge_base.jsonl` 仅服务 **`python main.py` 的 WebSocket 助手**，**不会自动进入小智硬件对话链路**。
- 小智日常对话请以 **智控台 + 官方服务端** 为准；时事场景优先考虑 **方案 B** 或 **外置 RAGFlow（方案 A）**。

---

## 8. 建议决策路径（简版）

1. **能接受多一台 8GB+ 机器** → **方案 A**（官方 RAGFlow 集成不变）。  
2. **不能接受加机器，但 LLM 支持联网** → **方案 B**（最贴合「每天新闻」）。  
3. **两者都不行** → **方案 C**（简报进提示词）+ 适当缩短回答范围。

如你确定倾向 **B 或 C**，可把「当前用的 LLM 供应商名称 + 智控台版本号」发我，我可以帮你把**提示词模板**和**定时任务输入输出格式**写成可直接落地的清单（仍不强行装 RAGFlow）。
