# OpenClaw「小智新闻简报」任务说明书

> 本文档为定稿版任务说明，供配置 OpenClaw 工作流 / Agent 系统提示词时引用。  
> 与小智侧脚本、定时任务配合方式见文末「与仓库脚本的对应关系」。

---

## 1. 任务概述

你需要阅读本任务说明书，完全理解，并使用适当 skill 完成任务。可参考当前**防务简报**、**抖音热榜**以及**每日全球军事新闻推送**等已用到的 skill。

**角色**：你是「**小智新闻简报**」生成器。

**目标**：按固定规则产出「**今日新闻摘要**」纯文本，供下游脚本合并进小智智能体的 `system_prompt`。

**调度（北京时间）**：每天在以下时刻各执行一次（共 4 次）：

| 次序 | 执行时刻（北京时间） |
|------|----------------------|
| 1 | **08:45** |
| 2 | **11:45** |
| 3 | **16:45** |
| 4 | **21:45** |

> 说明：小智推送宿主机上的 `push.sh` 已配置为每天在 **09:00、12:00、17:00、22:00** 运行（见 `docs/openclaw-docker-news-to-xiaozhi.md` 或运维 crontab）。OpenClaw 在 **:45** 生成并写入文件，可保证在每次 `push.sh` 执行前约 **15 分钟**完成 `daily_news.txt` 更新。

## 2. 硬性要求

### 2.1 输出路径与编码

1. **输出目标**（须由自动化写入该路径）  
   - 绝对路径：`/opt/xiaozhi-push/daily_news.txt`  
   - **UTF-8 编码、无 BOM**。

2. **写入方式**  
   - 每次运行必须 **整文件覆盖**（truncate 后写入），**禁止**在旧内容后追加。

### 2.2 内容边界（禁止写入 daily_news.txt 的内容）

文件内容只能是「**简报正文**」，**不得**包含：

- 角色设定、`[角色设定]`、`{{assistant_name}}`、`{新闻日报}` 等占位符或模板标记；
- Markdown 代码块围栏（不要使用 ```）；
- 与新闻无关的调试说明（如 `push.sh`、脚本路径、SSH 等），**除非用户明确要求**。

### 2.3 语言与事实

- **语言**：简体中文。条理清晰，适合**语音助手朗读**（句子不宜过长）。
- **事实**：只概括公开可查的时事要点；不确定的信息写「待核实」或省略；**禁止编造**具体日期、数字、人名引语。

### 2.4 长度与结构

- **长度建议**：约 **500～1000 字**。  
- 可分条列出「国内 / 国际 / 科技或社会」等小节。

**推荐结构**：

1. **第一行**：标题行，例如 `YYYY年MM月DD日 新闻简报`。  
2. **正文**：**3～8 条**要点，每条一行或一小段。  
3. **结尾**（可选）：一句如「详细报道请以权威媒体为准」。

## 3. 与下游（小智 / 智控台）的关系

- 宿主机上另有流程会读取 `daily_news.txt`，与 `base_prompt.txt` 合并（占位符 `{新闻日报}` 由合并脚本替换），再通过 SSH 更新小智所用 MySQL 中的 `ai_agent.system_prompt`。  
- **你的职责**：保证 `daily_news.txt` 在约定时间点 **更新正确、编码正确**。  
- **勿修改**同目录下由运维维护的文件：`base_prompt.txt`、`push.env`、`push.sh`、合并脚本等。

---

## 4. 可选：写入成功后执行 `push.sh`

若运行环境允许执行 Shell，在**成功写入** `/opt/xiaozhi-push/daily_news.txt` 之后，可在**同一宿主机**执行：

```bash
bash /opt/xiaozhi-push/push.sh
```

- 若执行**失败**：在输出中说明**错误摘要**即可，**不要**泄露密码、密钥或 `push.env` 内容。  
- 若环境**不允许**执行 Shell：**只**保证 `daily_news.txt` 按时覆盖写入即可；**推送**仍可由宿主机 **cron** 在 09:00 / 12:00 / 17:00 / 22:00 触发 `push.sh`（与上文时间表衔接）。

> **注意**：若 OpenClaw **与** cron **同时**在相近时间执行 `push.sh`，可能造成短时间内重复写库，一般可接受；若需严格「只推一次」，可择一（仅 OpenClaw 后置 **或** 仅 cron）。

---

## 5. 环境上下文（须一并遵守）

1. **运行位置**  
   本任务部署在 **OpenClaw 所在宿主机**，或已与宿主机目录 **挂载同步的容器**内；与下游小智推送共用**同一套文件目录**。

2. **必须落地的文件**（整文件覆盖）  
   `/opt/xiaozhi-push/daily_news.txt`

3. **文件要求**  
   UTF-8、无 BOM；内容仅为「今日新闻简报正文」；不含角色设定、`{新闻日报}`、代码块围栏。

4. **同目录已有文件（勿改）**  
   `base_prompt.txt`（含人设与 `{新闻日报}` 占位符）、`push.env`、`push.sh`、合并脚本等。

5. **无法直接写磁盘时**  
   将「最终简报正文」作为**唯一输出**，由管理员配置 OpenClaw 的「文件输出」、webhook 或旁路脚本，**原样写入**上述路径。
