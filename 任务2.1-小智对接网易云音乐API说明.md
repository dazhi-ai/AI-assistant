# 任务 2.1 · 小智对接网易云音乐 API 说明

> **目标**：让 ESP32 语音助手（小智）通过说话指令播放网易云音乐，支持按歌名搜索、按歌单名选择、按序号定位等多种方式。  
> **前提条件**：网易云 API（NeteaseCloudMusicApi）已部署且 Cookie 已配置完毕。

---

## 一、整体架构

```
用户说「播放七里香」/ 「播放第3个歌单第5首」
      ↓ ASR 识别
xiaozhi-esp32-server（Docker 容器）
      ↓ LLM function_call
  play_music(song_name, playlist_name, song_index)
      │
      ├─ 搜索模式（song_name 非空）
      │    └─ /cloudsearch 搜歌 → /song/url/v1 → 下载 MP3
      │
      └─ 歌单模式（playlist_name 非空）
           ├─ /user/playlist 获取账号所有歌单
           ├─ 模糊匹配歌单名（支持序号引用"第3个"）
           ├─ /playlist/track/all 取曲目列表
           ├─ 按 song_index 取指定曲或随机抽
           └─ /song/url/v1 → 下载 MP3
      ↓
NeteaseCloudMusicApi（Node.js，宿主机 3000 端口）
      ↓
网易云音乐服务器
      ↓
ESP32 设备播放
```

## 四、部署步骤

### 4.1 上传插件到服务器并注入容器

```powershell
# 开发机（Windows）→ 服务器
scp -P 1258 "d:\AI-assistant\scripts\xiaozhi\plugins_func\functions\play_music_netease.py" `
    root@<服务器IP>:/tmp/play_music_netease.py
```

```bash
# 服务器端
docker cp /tmp/play_music_netease.py \
    xiaozhi-esp32-server:/opt/xiaozhi-esp32-server/plugins_func/functions/play_music_netease.py

# 清除 Python 字节码缓存，防止旧代码生效
docker exec xiaozhi-esp32-server \
    find /opt/xiaozhi-esp32-server/plugins_func -name '__pycache__' -type d -exec rm -rf {} + 2>/dev/null

docker restart xiaozhi-esp32-server
```

### 4.2 配置 .config.yaml

在服务器 `/opt/xiaozhi-esp32-server/main/xiaozhi-server/data/.config.yaml` 中添加：

```yaml
plugins:
  play_music:
    netease_api_url: "http://172.17.0.1:3000"   # Docker 网关 IP，容器内访问宿主机
    netease_cookie: "MUSIC_U=xxxx; __csrf=yyyy"  # 网易云登录 Cookie
    netease_user_id: "你的网易云用户ID"            # 用于获取歌单列表
    fallback_music_dir: "./music"                # 网易云失败时的本地回退目录
    timeout: 15
```

> **⚠️ Docker 网络注意**：容器内无法用 `127.0.0.1` 访问宿主机服务。  
> 请使用 Docker bridge 网关地址（通常是 `172.17.0.1`）：  
> `ip route show | grep default` 在容器内确认。

### 4.3 确认 Intent 配置含 play_music

```yaml
Intent:
  function_call:
    type: function_call
    functions:
      - play_music    # ← 必须存在
      - get_weather
```

### 4.4 重启并验证

```bash
docker restart xiaozhi-esp32-server
docker logs -f xiaozhi-esp32-server
# 确认出现：当前支持的函数列表: ['play_music', ...]
```

## 六、插件核心设计

### 6.1 函数参数（三参数模式）

```python
play_music(
    conn,
    song_name: str = "",       # 歌曲名称，按名搜索时填写
    playlist_name: str = "",   # 歌单名或序号（"摇滚"/"3"/"我喜欢的音乐"）
    song_index: int = 0,       # 歌单第几首，0=随机（1-based）
)
```

LLM 根据 `play_music_function_desc` 描述决定传什么参数，以下是典型映射：

```
用户说："播放两只老虎"
  → song_name="两只老虎", playlist_name=""

用户说："播放我喜欢的歌"
  → song_name="", playlist_name="我喜欢的音乐"

用户说："播放摇滚歌单第3首"
  → song_name="", playlist_name="摇滚", song_index=3

用户说："播放第2个歌单"
  → song_name="", playlist_name="2"
```

### 6.2 歌单模糊匹配算法

`_fuzzy_match_playlist(playlists, keyword)` 按以下优先级匹配：

1. **序号引用**：关键词含数字或中文数字 → 返回第 N 个歌单
2. **特殊关键词**：「我喜欢」/「喜欢」/「liked」→ 返回第一个歌单
3. **完全匹配**：关键词 == 歌单名
4. **子串包含**：歌单名包含关键词
5. **反向包含**：关键词包含歌单名
6. **difflib 模糊**：相似度 ≥ 0.4（兜底，处理轻微 ASR 误识）

### 6.3 TTS 时序设计

```
play_music() 返回 Action.NONE（空 response）
      ↓  异步任务启动
_handle_netease_play()
  ①  FIRST(ACTION)          → 立即入队，抢占本轮
  ②  MIDDLE(TEXT: "正在查找...") → 立即入队，防止 TTS 1秒超时发 LAST
  ③  [网络请求：歌单/搜索]
  ④  MIDDLE(TEXT: "即将为您播放《xxx》") → 歌名确定后公告
  ⑤  [下载音频到 tmp/*.mp3]
  ⑥  MIDDLE(FILE: /path/to.mp3) → 音乐文件入队
  ⑦  LAST(ACTION)           → 结束本轮
```

> **关键**：步骤 ①② 必须在任何网络请求之前完成，否则 TTS 在 1 秒内收不到 TEXT 会超时自动发 LAST，导致设备切换到监听模式，后续音频无法播出。

### 6.4 版权失败自动换歌

歌单模式下，若某首歌 URL 为 null（版权限制），自动从同歌单随机换一首，最多重试 3 次：

```python
for _ in range(3):
    retry_info = _pick_song_from_playlist(songs, 0)  # 随机
    retry_url = _get_song_url(...)
    if retry_url:
        break
```

## 七、常见问题

### Q1. 歌单名 ASR 识别错误（如"王哥"→"网格"）

**原因**：ASR 对生僻词/英文混排词识别率低，声调近似的词会被误识。  
**解决**：
- 等小智读出歌单编号列表后，说「播放第N个歌单」（数字识别率高）
- 或将歌单改名为 ASR 友好的名称（纯中文常用词）

### Q2. 播放时无声音

检查日志中是否出现：
```
下载成功：/opt/xiaozhi-esp32-server/tmp/netease_xxx.mp3
音频已入队：...
添加音频文件到待播放列表：...
```
若下载成功但仍无声，可能是 TTS 时序问题，确认插件版本正确（最新版含占位 TEXT 机制）。

### Q3. Connection refused 连接网易云 API 失败

容器内无法访问宿主机 `127.0.0.1:3000`，改用 `172.17.0.1:3000`（见 4.2）。  
同时确认宿主机 Node.js 服务监听 `0.0.0.0`（非 `127.0.0.1`）：
```bash
# /etc/systemd/system/netease-api.service 中设置：
Environment=HOST=0.0.0.0
```

### Q4. URL 为 null（版权限制）

歌单模式会自动换歌重试 3 次。搜索模式失败则回退本地 `./music` 目录。

### Q5. Cookie 过期

Cookie 有效期约 180 天，过期后重新获取：
1. 浏览器登录 [music.163.com](https://music.163.com)
2. F12 → Application → Cookies → 复制 `MUSIC_U` 和 `__csrf`
3. 更新插件文件顶部 `_NETEASE_COOKIE` 常量，重新部署

### Q6. 语法错误导致服务无法启动

f-string 中不能包含与外层相同的引号（Python 3.10 限制）。写中文提示语时，避免在双引号 f-string 内用 ASCII 双引号，改用单引号或去掉引号。
